实现方式

Puma framework的用户管理模块如下图所示，该模块提供对系统用户的新增、修改、批量删除、批量重设密码、数据刷新以及翻页功能。
前端采用HTML+CSS+JQuery+JEasyui实现。
后台登录采用Spring framework + Spring Data + Hibernate实现。

新增用户

1. 点击用户管理界面中的新增按钮，弹出新增用户的表单窗口。

2. 新增用户的表单窗口有5个必填项，用户名、用户邮箱、密码、重复密码以及用户角色。如果不输入，会有相应的提示。
  • 用户名是用来在系统中显示的名称。
  • 用户邮箱是用户登录所使用的账号。邮箱格式必须正确，否则提供验证错误的信息。
  • 密码是用户登录所使用的密码。
  • 是否启用复选框是表示该用户账号是否可用。如果不可用，用户在登录的时候会提示用户名或密码错误。此处的处理方式有待修改，应该提示：用户不存在！
  • 是否管理员复选框，如果勾上该选择框，用户角色下拉框将会不可见，表示该用户拥有系统的所有权限。
  • 角色是分配给用户的角色。角色定义请参考角色管理的使用说明。
3. 用户输入完整信息后，点击"保存"按钮，如果所有表单元素正确，提示保存进度信息，保存成功后，提示保存成功，并且所添加的用户记录在当前表单的最上方，页面不跳转。

编辑用户

1. 选择要编辑的用户记录，点击用户管理界面中的"编辑"按钮，弹出编辑用户的表单窗口。

2. 鼠标双击要编辑的用户记录，弹出编辑用户的表单窗口。

3. 用户编辑的表单自动加载所选择用户的相关信息，密码信息除外。

4. 如果没有选择用户，就点击了"编辑"按钮，需要提示用户先选择用户后，再进行编辑工作。

5. 如果用户选择了多个用户，则编辑的是所选用户中第一个。

6. 用户修改完信息后，点击"保存"按钮，如果所有表单元素正确，提示保存进度信息，保存成功后，提示保存成功，并且所编辑的用户记录在当前表单原位置，页面不跳转。

删除用户

1. 选择要删除的用户记录，点击用户管理界面中的"删除"按钮，弹出确认窗口。

2. 点击"确认"按钮，提示删除进度信息，删除成功后，提示删除用户成功，页面不跳转

3. 点击"取消"按钮，确认窗口消失。

重设用户密码

1. 选择要重设密码的用户记录，点击用户管理界面中的"重设密码"按钮，弹出确认窗口。

2. 点击"确认"按钮，提示操作进度信息，重设密码成功后，提示重设用户密码成功，页面不跳转

3. 点击"取消"按钮，确认窗口消失。

刷新表格

1. 点击用户管理界面中的"刷新"按钮，表格中间出现加载进度信息，加载成功后消息。

分页操作

1. 表格每页默认显示20条记录。当操作20条的时候，自动分页。

2. 用户可以选择每页所显示的记录数。

3. 用户可以通过输入页码，跳转到指定页。

4. 用户可以进行上一页、下一页、第一页、最后一页的操作。

5. 显示记录总数和当前页。
